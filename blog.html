We can save any small data in the browser through local storage. And in the same way, we can save any small data in the browser through session storage.

But an important difference between them is that when we save any data in local storage, even if we close the browser or turn off the machine, we can access that data even if we enter the browser later. Unless we clear it manually.

On the other hand, in the case of session storage, we can save only one session amount of data in the browser. The data will be saved in the session storage until we close the browser or until then. They will be deleted immediately when we close the browser.

The event loop is a fundamental part of the JavaScript runtime that allows for non-blocking operations.

The event loop is a fundamental part of most modern programming languages and frameworks, including JavaScript and Node.js. Its primary function is to manage the execution of asynchronous code by monitoring a queue of events, and handling them in a non-blocking way.

Here's how the event loop works:

  • The event loop starts when our application starts running.
  • When our application encounters an asynchronous operation, such as a network request or a timer, it initiates that operation and then moves on to the next task.
  • While the asynchronous operation is happening in the background, our application can continue to execute other tasks in the meantime.
  • Once the asynchronous operation is complete, it adds the corresponding task to the task queue.
  • The event loop continuously checks if there are any pending tasks in the task queue.
  • If there are, it takes the first task in the queue and executes it.
  • If the task is synchronous, it runs immediately and the event loop moves on to the next task.
  • If the task is asynchronous, such as a callback function or a Promise, it initiates the operation associated with that task and moves on to the next task.
